# TASK3 双均线策略回测看板 Spec

## 1. 目标

在 `task3/` 目录下实现一个双均线策略回测看板，用于导入股票日线数据、配置双均线交易规则和风控/仓位/成本参数，执行回测，并输出图表、交易明细、绩效指标和可导出的报告。

本文件最初为执行前 spec；用户已确认第 21 节口径，可据此进入实现阶段。

## 2. 输出目录

所有 TASK3 产物统一放在：

- `task3/`

确认后计划生成：

- `task3_double_ma_backtest_dashboard_spec.md`：本 spec 文件。
- `double_ma_backtest_dashboard.html`：交互式回测看板主页面。
- `double_ma_backtest_engine.py`：可复现回测核心逻辑。
- `double_ma_backtest_readme.md`：使用说明和字段说明。
- `sample_data/`：示例数据，可复用 TASK1 的 A 股、港股前复权日线数据。
- `reports/`：导出的 HTML / PDF 回测报告输出目录。
- `exports/`：交易明细、净值曲线等 CSV 导出目录。

首版建议实现为“本地静态 HTML + 浏览器内计算”，保证可以直接打开使用；如 PDF 导出效果不稳定，再补充本地 Python 报告生成脚本。

## 3. 数据输入与质量判断

### 3.1 数据来源

首版支持本地 CSV 导入。

建议内置示例数据：

- `task1/smic_a_daily_qfq.csv`
- `task1/smic_hk_daily_qfq.csv`

后续可扩展为输入股票代码并通过本地后端或 MCP 动态获取数据。

### 3.2 必需字段

内部标准字段：

- `trade_date`：交易日期。
- `open`：开盘价。
- `high`：最高价。
- `low`：最低价。
- `close`：收盘价。
- `vol`：成交量。

可选字段：

- `ts_code`：证券代码。
- `name` / `stock_name`：证券名称。
- `pre_close`：前收盘价。
- `amount`：成交额。
- `adj_factor`：复权因子。
- `qfq` / `adj_type` / `adjusted` 等复权标识字段。

如果缺少 `pre_close`，可用前一交易日 `close` 补齐，但需在质量提示中说明。

### 3.3 数据质量检查

导入后必须显示数据质量提示：

- 字段完整性：必需字段是否存在。
- 日期有效性：日期是否可解析，是否有重复日期。
- 排序情况：是否已按日期升序，若不是则自动排序并提示。
- 缺失值情况：核心 OHLCV 字段缺失数量。
- 价格合理性：`high >= low`、`high >= open/close`、`low <= open/close`。
- 成交量合理性：成交量是否缺失、为负或长期为 0。
- 样本范围：起止日期和交易日数量。
- 数据频率：首版默认日线数据；若检测到非日线频率，提示但不继续推断。

### 3.4 是否复权判断

复权状态只能做“提示和风险说明”，不能完全保证自动识别正确。

判断逻辑：

- 如果存在明确字段 `adj_factor`、`adj_type`、`qfq`、`adjusted`，优先读取字段并展示。
- 如果文件名包含 `qfq`、`hfq`、`adj`、`复权`，给出“疑似复权数据”提示。
- 如果出现较大的异常跳空，例如单日 `abs(close / pre_close - 1)` 超过 30%，提示可能存在未复权、除权除息或停复牌跳空影响。
- 如果无法判断，展示“复权状态未知，回测信号和收益可能受分红送转影响”。

## 5. 回测基础参数

左侧基础参数包括：

- 回测开始日期。
- 回测结束日期。
- 初始资金，默认 `100,000`。
- 交易价格口径，首版默认使用下一交易日开盘价成交；可选扩展为当日收盘价成交。
- 股票最小交易单位：
  - A 股默认 100 股一手。
  - 港股可设置每手股数，默认 1 或按用户输入。

建议首版默认：

- 信号在第 `t` 日收盘后产生。
- 买入/卖出在第 `t+1` 个交易日开盘价执行；如果中间是周末或休市日，则顺延到下一个有数据的交易日。
- 如果已到最后一个交易日，未执行的信号只记录，不形成成交。

## 6. 双均线策略规则

### 6.1 参数

- 短期均线周期 `short_window`，默认 10。
- 长期均线周期 `long_window`，默认 30。
- 必须满足 `short_window < long_window`。
- 均线类型首版使用简单移动平均线 `SMA`。

### 6.2 买入信号

基础版本：

- 金叉买入：昨日 `short_ma <= long_ma`，今日 `short_ma > long_ma`。

如果启用成交量过滤、趋势过滤、最大回撤等待恢复等条件，则所有买入条件必须同时满足才允许买入。

### 6.3 卖出信号

任一条件触发即卖出：

- 死叉：昨日 `short_ma >= long_ma`，今日 `short_ma < long_ma`。
- 固定比例止损。
- ATR 止损。
- 移动止损。
- 最大回撤控制触发后的强制降仓或清仓规则。

卖出不受成交量过滤限制。成交量过滤只约束买入。

## 7. 成交量过滤

该模块可勾选启用。

参数：

- 成交量均线周期 `volume_window`，默认 20。
- 成交量放大倍数阈值 `volume_multiplier`，默认 1.5。

计算：

- `volume_ma = 最近 N 日成交量平均值`。
- `volume_ratio = 当日成交量 / volume_ma`。

买入过滤条件：

- 当前成交量不为空。
- `volume_ma` 不为空。
- `vol >= volume_ma`。
- `volume_ratio >= volume_multiplier`。

只有以上条件同时满足，金叉买入信号才允许转为实际买入信号。

## 8. 仓位管理

该模块可勾选启用。未启用时，默认每次买入使用全部可用现金，但仍受最大仓位上限约束。

### 8.1 共同参数

- 最大仓位上限，默认 100%，范围 0-100%。
- 是否允许持续加仓，默认允许。
- 加仓间隔 N 个交易日，默认 5。
- 最小现金保留比例，可选，首版默认 0。
- 交易数量按最小交易单位向下取整。

仓位上限定义：

- `持仓市值 / 当前总资产 <= 最大仓位上限`。

### 8.2 固定金额

参数：

- 初始买入金额。
- 持续加仓金额。

默认值：

- 初始买入金额默认 `30,000`。
- 持续加仓金额默认 `10,000`。

规则：

- 首次建仓使用初始买入金额。
- 已持仓且触发加仓信号时，如允许加仓，则使用持续加仓金额。
- 实际买入金额不得超过可用现金和最大仓位上限。

### 8.3 固定比例

参数：

- 初始买入比例。
- 持续加仓比例。

规则：

- 首次建仓使用 `当前总资产 * 初始买入比例`。
- 已持仓且触发加仓信号时，使用 `当前总资产 * 持续加仓比例`。
- 实际买入金额不得超过可用现金和最大仓位上限。

### 8.4 静态凯利公式

首版只实现静态凯利，不根据后续交易动态滚动更新。

参数：

- 凯利使用比例：全凯利、半凯利、四分之一凯利、自定义比例。
- 至少完成多少笔交易后才能使用凯利，默认 20。
- 凯利启用前的固定初始仓位比例，默认 30%。
- 最大仓位上限。

静态凯利计算：

- `kelly_fraction = win_rate - (1 - win_rate) / payoff_ratio`
- `payoff_ratio = 平均盈利交易收益率 / abs(平均亏损交易收益率)`

首版可采用两种输入方式之一，建议优先实现手动输入：

- 手动输入预估胜率和盈亏比，用于静态凯利。
- 或使用前 `M` 笔已完成交易估计胜率和盈亏比，完成交易数不足 `M` 时使用固定初始仓位比例。

风险约束：

- 若 `kelly_fraction <= 0`，则本次买入仓位为 0，不买入。
- 实际凯利仓位 = `kelly_fraction * 凯利使用比例`。
- 最终仓位不得超过最大仓位上限。

## 9. 风险控制

该模块可勾选启用。风险控制卖出优先级高于普通死叉卖出。

### 9.1 固定比例止损

参数：

- 止损比例，默认 8%。

规则：

- `当前价格 <= 买入均价 * (1 - 止损比例)` 时触发卖出。

### 9.2 ATR 止损

参数：

- ATR 周期，默认 14。
- ATR 倍数，默认 2。

计算：

- `TR = max(high - low, abs(high - pre_close), abs(low - pre_close))`
- `ATR = TR` 的 N 日均值。
- `止损价 = 买入价 - ATR倍数 * ATR`

规则：

- 当前价格跌破 ATR 止损价时触发卖出。
- 如果 ATR 不可用，则该日不触发 ATR 止损，并在交易原因中记录 ATR 不足。

### 9.3 移动止损

参数：

- 回落幅度阈值，默认 10%。

计算：

- `持仓期间最高价 = 建仓后至当前日最高价的最大值`。
- `回落幅度 = 当前价格 / 持仓期间最高价 - 1`。

规则：

- `回落幅度 <= -回落幅度阈值` 时触发卖出。

### 9.4 最大回撤控制

参数：

- 最大回撤阈值，默认 20%。
- 等待净值恢复规则：
  - 方式 A：净值恢复到历史高点的指定比例后才允许重新买入，默认 95%。
  - 方式 B：等待固定交易日数后才允许重新买入，默认 20 日。
- 触发后动作：
  - 首版建议清仓并暂停新买入。

计算：

- `drawdown = 当前净值 / 历史最高净值 - 1`。

规则：

- 当 `drawdown <= -最大回撤阈值` 时触发最大回撤控制。
- 若当前有持仓，按下一交易日开盘价清仓。
- 在恢复条件满足前，禁止新的买入信号转为成交。

## 10. 交易成本设置

参数：

- 手续费率，默认 0.03%。
- 最低手续费，默认 0。
- 卖出印花税，默认 0.05% 或由用户设置。
- 滑点模拟，默认 0.02%。

买入成交价：

- `buy_price = raw_price * (1 + slippage_rate)`。

卖出成交价：

- `sell_price = raw_price * (1 - slippage_rate)`。

交易成本：

- 买入成本 = `成交金额 * 手续费率`，不低于最低手续费。
- 卖出成本 = `成交金额 * 手续费率 + 成交金额 * 印花税率`，手续费不低于最低手续费。

交易明细中必须记录每笔成交的滑点影响、手续费、税费和总成本。

## 11. 趋势过滤

该模块可勾选启用。

参数：

- 长期趋势均线周期 `trend_window`，默认 60。

买入过滤条件：

- `close > trend_ma`。

趋势过滤只约束买入，不约束卖出。

## 12. 回测撮合与持仓规则

### 12.1 时间顺序

每个交易日按以下顺序处理：

1. 计算当日指标和信号。
2. 根据已有持仓判断止损、移动止损、最大回撤控制和死叉卖出信号。
3. 根据信号决定下一交易日开盘执行的订单。
4. 在下一交易日按开盘价、滑点和手续费撮合。
5. 更新现金、持仓数量、持仓市值、总资产和净值。

为降低复杂度，首版可采用“信号日收盘判断，下一交易日开盘成交”的统一口径。这里的下一交易日指数据中的下一条交易记录；如果信号日后遇到周末或休市日，会顺延到下一个有数据的交易日。

### 12.2 多个卖出原因同时出现

若同一天多个卖出条件同时出现，交易原因按优先级展示：

1. 最大回撤控制。
2. 固定比例止损或 ATR 止损。
3. 移动止损。
4. 死叉。

交易明细中仍需记录所有同时触发的条件。

### 12.3 重复买入和加仓

- 空仓时，买入信号触发建仓。
- 已持仓时，不再使用“再次金叉”作为加仓条件。
- 加仓条件为：短均线高于长均线，且短均线和长均线当天都比前一交易日上行。
- 两次加仓信号之间必须至少间隔 N 个交易日，N 由左侧参数设置，默认 5。
- 具体判断口径：
  - `short_ma[t] > long_ma[t]`
  - `short_ma[t] > short_ma[t-1]`
  - `long_ma[t] > long_ma[t-1]`
  - `t - 上次加仓信号日索引 >= N`
- 如果允许持续加仓，则按当前仓位管理方式计算加仓预算。
- 如果不允许持续加仓，已持仓时忽略加仓信号，但在信号表中记录“已有持仓且禁止加仓”。

## 15. 绩效指标

### 15.1 收益类指标

- 总收益率：`最终总资产 / 初始资金 - 1`。
- 年化收益率：`(最终总资产 / 初始资金) ** (252 / 交易日数) - 1`。
- 累积回报：净值曲线相对初始值的累计收益。
- 基准收益率：同期买入并持有收益。
- 超额收益：`策略总收益率 - 基准收益率`。

### 15.2 风险类指标

- 最大回撤：净值曲线相对历史高点的最大跌幅。
- 最大回撤开始日期。
- 最大回撤结束日期。
- 最大回撤修复日期，如样本内未修复则显示未修复。

### 15.3 交易质量类指标

- 交易次数。
- 完整闭合交易次数。
- 胜率：盈利闭合交易数 / 闭合交易数。
- 平均盈利。
- 平均亏损。
- 盈亏比：平均盈利 / abs(平均亏损)。
- 平均持仓天数。

### 15.4 综合类指标

- 夏普比率：`年化平均日收益 / 年化日收益波动`。
- 默认无风险收益率为 0，可在参数区扩展设置。
- 若日收益波动为 0，则夏普比率显示为空并提示不可计算。

## 18. 交互与校验

执行前校验：

- 必须已导入数据。
- 必须选择有效回测区间。
- 起始日期早于结束日期。
- 初始资金大于 0。
- 短均线周期小于长均线周期。
- 所有比例参数在合理范围内。
- 手续费、税费、滑点不能为负。
- 最大仓位上限在 0-100%。

运行状态：

- 参数错误时阻止回测并给出明确提示。
- 数据不足以计算均线时阻止回测。
- 指标前期空值不应产生交易信号。
- 每次回测后锁定一份参数快照，用于报告导出。

## 19. 实现建议

### 19.1 前端

建议使用：

- 原生 HTML/CSS/JavaScript。
- Plotly.js 绘制交互图表。
- PapaParse 解析 CSV。

理由：

- 当前作业已有静态 HTML 工具。
- 用户可直接打开 HTML 使用。
- 不依赖复杂前端构建环境。

### 19.2 回测引擎

建议将回测逻辑写成清晰的纯函数模块：

- 数据清洗。
- 指标计算。
- 信号生成。
- 仓位计算。
- 订单撮合。
- 账户曲线更新。
- 指标统计。
- 报告导出。

即使首版在浏览器中实现，也应保持函数边界清晰，便于后续迁移到 Python 或后端。

## 20. 验收标准

确认实现完成后，需要满足：

- 能导入标准日线 CSV 并展示数据质量和复权提示。
- 能设置回测区间、初始资金、双均线参数和交易成本。
- 能启用/关闭成交量过滤、仓位管理、风险控制和趋势过滤。
- 金叉买入、死叉卖出逻辑正确。
- 止损、ATR 止损、移动止损、最大回撤控制中任一触发即可卖出。
- 买入受成交量过滤和趋势过滤约束，卖出不受过滤约束。
- 能绘制价格均线交易信号图。
- 能绘制净值、持仓、回撤图。
- 能展示交易明细、信号日志、绩效指标和最终仓位。
- 能导出 HTML 报告。
- PDF 报告可通过自动导出或打印 HTML 获得。

## 21. 待用户确认点

以下口径已确认：

1. 首版采用“信号日收盘判断，下一交易日开盘成交”。
2. 首版只做单股票单标的回测，不做组合回测。
3. 凯利公式不手动输入预估胜率和盈亏比，采用前 `M` 笔已完成交易估计；交易数不足 `M` 时使用固定初始仓位比例。
4. 最大回撤控制触发后按“清仓并暂停新买入”处理。
5. PDF 导出接受“生成可打印 HTML，再由浏览器打印为 PDF”作为首版方案。
